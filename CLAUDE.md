# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

An educational repository that teaches **harness engineering** for AI agents by deconstructing Claude Code's architecture. The core philosophy: "The model IS the agent. The code is the harness."

Two main components:
- **`agents/`** — 12 progressive Python sessions (s01–s12) + a capstone (`s_full.py`), each adding one harness mechanism
- **`web/`** — Next.js interactive learning platform with i18n (en/zh/ja)

## Common Commands

### Python (agents)

```bash
pip install -r requirements.txt        # anthropic + python-dotenv
cp .env.example .env                   # then set ANTHROPIC_API_KEY and MODEL_ID

python agents/s01_agent_loop.py        # run a single session
python agents/s_full.py                # run the capstone (all mechanisms)
```

### Web platform

```bash
cd web
npm install
npm run dev                            # starts dev server (runs extract first)
npm run extract                        # extract session content from Python files
npx tsc --noEmit                       # type check
npm run build                          # production build (static export)
```

### Tests (CI)

```bash
python tests/test_unit.py                    # unit tests
python tests/test_v0.py                      # session test (v0–v9, v8a–v8c)
```

Note: the `tests/` directory is referenced in CI but not currently present in the repo.

## Architecture

### Session Progression

Each session adds exactly ONE mechanism without changing the core agent loop:

| Phase | Sessions | Mechanisms |
|-------|----------|------------|
| Loop | s01–s02 | Core while-loop + tool dispatch map |
| Planning | s03–s06 | TodoManager, subagents (isolated `messages[]`), on-demand skill loading, 3-layer context compression |
| Persistence | s07–s08 | File-based task CRUD with dependency graph, daemon thread background tasks with notification queues |
| Teams | s09–s12 | JSONL mailbox teammates, shutdown/approval FSM protocols, idle-cycle auto-claiming, worktree directory isolation + event logging |
| Capstone | s_full | All mechanisms combined |

### Core Pattern (every session builds on this)

```
while True:
    response = client.messages.create(...)
    if response.stop_reason != "tool_use": return
    execute tools → append tool_results → continue loop
```

### Key Directories

- `agents/` — Python implementations, ~100–800 lines each, self-contained
- `docs/{en,zh,ja}/` — Mental-model-first markdown docs, one per session
- `skills/` — Domain knowledge modules (SKILL.md files) loaded on-demand by s05+
- `web/src/app/[locale]/` — Next.js App Router with locale-based i18n routing
- `web/src/components/` — React components including diff viewers, simulators, visualizations

### Web Stack

Next.js 16 (App Router, static export) · React 19 · TypeScript · Tailwind CSS 4 · Framer Motion · remark/rehype for markdown · diff library for code comparisons

Deployed to Vercel at `learn.shareai.run`.

## Environment Configuration

Set via `.env` (see `.env.example`):
- `ANTHROPIC_API_KEY` — required
- `MODEL_ID` — e.g. `claude-sonnet-4-6`, `claude-opus-4-6`
- `ANTHROPIC_BASE_URL` — optional, for compatible providers (MiniMax, GLM, Kimi, DeepSeek)

## CI Pipelines

- **ci.yml**: TypeScript typecheck + Next.js build (Node 20)
- **test.yml**: Python unit tests + parameterized session tests (Python 3.11) + web build

## Runtime Artifacts (gitignored)

`.tasks/`, `.teams/`, `.transcripts/` — generated by agents at runtime; safe to delete.
